How to Install Software

Clone Repository

In Visual Studio Code, open a terminal and use the following command:

git clone https://github.com/Bmfioresi/stage-managers-book.git

Install Node.js and Node Package Manager (npm)

Follow the instructions here to install Node.js and npm.

Install the Dependencies

Inside both the client and server directories, run the command npm install to install the respective dependencies outlined in each package.json. This will generate a folder called node_modules, which is excluded from the git repository because its size is unncessarily large and annoying to download/upload with git.

In case of dependency errors, try the command npm install --save --legacy-peer-deps; this makes the installation less restrictive.

Google Sign-In/OAuth 2.0 Incorporation

To effectively incorporate Google Sign-In and OAuth 2.0 in our application, it is crucial to start by setting up a Google Cloud project, enabling essential APIs such as Google Calendar API and Gmail API, and configuring OAuth consent screens to inform users about their data access. This setup includes obtaining credentials like the Client ID and Client Secret from the Google Cloud Console and ensuring the redirect URIs are correctly configured for handling OAuth callbacks in our Node.js backend. The Google Sign-In button is integrated using the Google Identity Services library within our React components, managing authentication responses to extract and handle access tokens securely. These tokens are essential for interacting with Google services, emphasizing the importance of appropriately managing token scopes and handling token expiration. A significant issue we had with implementing this was handling the Cross-Origin Opener Policy (COOP) errors. Addressing these errors is crucial; typically, these are mitigated by setting correct COOP headers in our Node.js server responses to prevent typical cross-origin issues. However, we were not able to squash this issue.

For advanced functionality, such as modifying Google Calendar or sending automated emails via Gmail, developers should use the access token with Google APIs, facilitated by secure API call handling in our Node.js backend. Developers must also implement best practices for the safe storage and transmission of tokens to protect user information and prevent unauthorized access. Maintaining a detailed changelog of OAuth setup modifications and staying updated with the latest Google API documentation and OAuth 2.0 standards may assist in troubleshooting and future updates.

Forgot Password Functionality

The forgot password feature in our application intends to allow users to reset their passwords securely. We have developed a page for forgotten passwords on the front end using React. This page collects the user’s email address and sends it to our backend through an API call. In the backend commented out, we handle this request by generating a secure, temporary link that allows the user to reset their password. This link intends to be sent to the user’s email address using the Gmail account associated with this project.

For email functionality, we utilize Node.js, which has the nodemailer library configured to send emails via Gmail. However, the email was not sent successfully. Future developers may need to check several vital areas to mitigate this: first, ensure that the correct Gmail API settings are enabled in the Google Cloud Console and that the OAuth credentials are correctly set up. Verifying that the Gmail account has “less secure apps” access enabled is essential. Additionally, ensure that the SMTP configuration in nodemailer is correct, including the host, port, and authentication details.

If problems persist, developers should consult the Gmail API documentation and check the Node.js logs for any error messages that could indicate what may be going wrong. Possible issues could include network problems, incorrect API scopes, or Google blocking sign-ins from new devices or locations it deems suspicious.

How to Reduce Potential Vendor Lock-In

Reducing dependency on MongoDB Atlas and avoiding vendor lock-in involves considering local data management solutions and understanding the implications of database standardization:

  • Understanding Database Standards: NoSQL databases, like MongoDB, do not adhere to a unified query standard, which complicates the development of a vendor-agnostic platform. MongoDB utilizes a document- oriented approach.
  • Exploring Abstraction Layers: Although libraries and frameworks provide abstraction for basic CRUD operations across different databases, they might not fully cover all the specific features of each database type. This can limit their effectiveness in production, especially for complex queries and operations.
  • Local MongoDB Setup: As a practical workaround, consider periodically downloading your project data from MongoDB Atlas and storing it locally on Docker. This approach reduces reliance on Atlas and ensures that you have a backup in case of changes in service terms or costs. You can export data from MongoDB in BSON format and use it to set up a local MongoDB instance. Ensure your Docker environment is configured to handle this data securely and efficiently.
  • Seeking Further Guidance: Continue seeking advice and feedback from instructors or experienced developers. Their insights could provide valuable shortcuts or alternatives you might have yet to consider.

Frequently Asked Questions

How do I create an account?

To create an account, simply click on the "Don't have an account? Sign-up" link on the sign-in page below the "Sign-in" button. This will redirect you to the account creation page.

You will need to provide your full name, email address, and a password. Ensure that the password meets our specified criteria and is entered twice for confirmation.

Your password must meet certain criteria to ensure security. It includes a minimum of eight characters in length, a mix of uppercase and lowercase letters, and at least one number and special characters. This criteria is outlined on the account creation page.

How do I create a new hub for my production?

Once logged in, navigate to the side panel and select “Create New Hub.” Enter the required information about the production, such as name and description. After creating the hub, you can invite cast and crew members by sending the access code to their emails, allowing them to join and participate in the fun!

Are there any restrictions on uploading scripts and images to a hub?

Yes, files must be in supported formats (e.g., PDF for scripts, JPEG or PNG for images) and should not exceed the size limit of 25MB per file. Make sure your files are correctly formatted and named before uploading. If you encounter errors during upload, check your internet connection, file format, and file size and try again.

How can I update my user profile?

After logging in, you can update your profile information by navigating to the page. Scroll to the bottom and click the “Edit Profile” button. You can change details such as your contact information, bio, and profile picture. Make sure to save changes before exiting the page.
